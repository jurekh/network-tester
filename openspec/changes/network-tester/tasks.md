# Tasks

Execution is organized into stages. Each stage delivers a vertical slice of the solution together with the testbed increment that verifies it end-to-end, and ends with a verification gate. Do not start a stage before the previous stage's gate passes. Verification levels per stage: unit tests (mocked subprocess/API), integration tests (ops Harness, jubilant), and end-to-end runs in the nested-virtualization testbed (design D18). The testbed lives entirely inside one outer LXD VM (`nt-testbed`); the host needs only LXD, uv, and charmcraft, and `testbed down` removes all testbed state from the machine.
